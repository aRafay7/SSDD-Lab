Welcome to the Secure Flask Application

Lab-07: Implementing 5 Security Practices

Security Features Implemented:
  • 1. Secure Input Handling
    Forms use Flask-WTF validators to prevent SQL injection and XSS attacks. All input is validated before processing.
  • 2. Parameterized Queries
    SQLAlchemy ORM automatically uses parameterized queries to prevent SQL injection attacks.
  • 3. CSRF Protection & Session Management
    CSRF tokens in all forms, secure HTTP-only cookies, and proper session timeout configuration.
  • 4. Secure Error Handling
    Custom error pages that don't expose sensitive information like stack traces or file paths.
  • 5. Secure Password Storage
    All passwords are hashed using bcrypt before storage in the database.
